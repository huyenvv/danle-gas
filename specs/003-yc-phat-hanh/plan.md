# Implementation Plan: YC Phát hành

**Branch**: `003-yc-phat-hanh` | **Date**: 2026-05-31 | **Spec**: [spec.md](spec.md)

**Input**: Feature specification from `specs/003-yc-phat-hanh/spec.md`

## Summary

Add "YC Phát hành" workflow action from "Chờ duyệt" status (alongside Giao việc, Từ chối, Lưu trữ). GĐ reviews a doc and requests creator to publish it directly. Mirrors the "Từ chối" pattern: GĐ enters a reason, document transitions to "YC Phát hành", creator sees publish button (no edit), email notification sent. All publishers (VT + users with canCreate + canPublish, excluding admin) lose edit capability on "YC Phát hành" status — only creator sees publish button. Admin is exempt from restrictions. Reuses "Lý do từ chối" field for the reason and adds a dedicated email template tab in Settings.
